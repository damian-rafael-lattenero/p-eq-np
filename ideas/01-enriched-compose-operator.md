# The Enriched Composition Operator

## Classical vs Enriched

```
Classical:    (.) :: (b -> c) -> (a -> b) -> (a -> c)
              f . g = \x -> f (g x)
              -- No information propagated. "Naked" functions.

Enriched:     (⊗) :: EA m b c -> EA m a b -> EA m a c
              EA m2 f ⊗ EA m1 g = EA (m1 <> m2) (f . g)
              -- Metadata combines monoidally. "Rich" functions.
```

## What Changes

| Aspect | Classical (.) | Enriched (⊗) |
|--------|--------------|---------------|
| Information flow | Only through values | Through values AND metadata |
| Inspection | Must execute to observe | Can inspect metadata without executing |
| Associativity | f . (g . h) = (f . g) . h | Same, PLUS metadata associativity |
| Identity | id | EA mempty id |

## Key Property

The metadata of a composed path is the monoidal sum of all individual metadata.
For Subset Sum with metadata = (Sum Int):

```haskell
include 3 ⊗ skip 7 ⊗ include 5 ⊗ include 2
-- metadata = Sum 3 <> Sum 0 <> Sum 5 <> Sum 2 = Sum 10
-- You know this path sums to 10 WITHOUT running it on any input
```

## The Operator as a Functor

The enriched composition can be seen as a functor from the free category
generated by the decision graph to the enriched category. This functor
"decorates" each path with its sum.
